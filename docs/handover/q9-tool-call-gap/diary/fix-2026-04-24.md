## Scope

Ship the Option A fix from the diagnosis — flatten the `oneOf`
discriminated-union schemas that chrome-devtools-mcp exposes for its three
compound tools (`interact`, `observe`, `trace`) before handing them to
Ollama's `/v1/chat/completions`. Gemma 4 E4B cannot template `oneOf` in its
tool-call template, so without this transformation every calibration
trajectory terminates at turn 1 with the args emitted to
`message.content` — the signature of the 2026-04-24 baseline's uniform 25%
score floor across 60 runs.

Secondary cleanup falls out with the fix: the `repairAndParseJson` regex
helper in `tool-loop.ts` was a workaround for malformed JSON on the
content channel that no longer happens once `tool_calls` flows correctly,
and the `DEFAULT_NUM_CTX = 32768` cap in `ollama-client.ts` was 25% of
Gemma 4's native 131072 window.

## Deviations from Option A

None of substance. The helper at
`packages/local-agent/src/mcp-bridge.ts` implements exactly the shape
described in the diagnosis: `command` becomes a required top-level enum,
per-variant properties are hoisted to the top level as optional, and
`required` at the top level contains only `command`.

Minor choices made along the way (none of which change the contract):

- **Description merging.** When two variants declare the same hoisted
  property with distinct descriptions (e.g. `filePath` appears on
  `observe.snapshot`, `observe.screenshot`, `observe.network.requestFilePath`…),
  descriptions are merged with `" / "`. If descriptions match, the
  duplicate is dropped.
- **Prop-name collisions with different types.** `interact.type.text` is
  `{ type: "string" }` and `interact.wait_for.text` is `{ type: "array", items: { type: "string" }, minItems: 1 }`.
  First-seen variant wins; the MCP server validates at call time and
  ignores the wrong shape. The diagnosis already flagged this as
  tolerable ("MCP server ignores unknown fields") and the unit test pins
  the behavior.
- **Pass-through guard.** Schemas whose top-level shape doesn't match the
  discriminated-union pattern — the 5 flat tools (`click`, `fill`, `hover`,
  `select`, `wait_for`) and `select.option`'s nested `anyOf` — are
  returned by identity, so the helper is a no-op for them. Unit test
  pins this too.
- **Wrapper detection order.** `detectWrapperKey(tool.inputSchema)` still
  runs on the ORIGINAL (pre-flatten) schema at line 91, so the
  call-time auto-wrap keeps rebuilding `{ action: { command, ...args } }`
  for the MCP server. This is essential — the flattened schema has no
  `action` property, but the underlying MCP tool still requires it at
  call time. Verified by the `detectWrapperKey` unit tests in the new
  suite.

## Ambiguity hit during implementation

- **How to import `flattenOneOf` from the post-fix probe.** Team-lead's
  hint suggested either building `@neuve/local-agent` or importing the
  `.ts` directly. Local-agent's `vp pack` build bundles everything into a
  single `dist/main.js` that starts the ACP agent on stdin/stdout — the
  helper is not individually importable from the build. Rather than
  introduce a TS runtime (tsx / jiti) just for this probe, the helper was
  duplicated inline in the probe script with a pointer to the
  authoritative unit test at
  `packages/local-agent/tests/flatten-one-of.test.ts`. Risk: drift between
  the probe's inline copy and the shipping helper. Mitigation: if a future
  change to the bridge helper breaks the flatten contract, the unit test
  will catch it first; if someone wants to re-run the probe after such a
  change, they re-copy the helper (< 40 lines).
- **Whether to commit the `*-request-body.json` / `*-response.json` files
  from the post-fix probe run.** Task-5 description explicitly bans
  committing response artifacts ("DO NOT commit the saved response
  files"). Left them uncommitted — only `probe-b-post-fix.mjs` itself is
  versioned.
- **Pre-existing formatting drift** in `src/agent.ts`, `src/ollama-client.ts`,
  and `package.json` was picked up by `pnpm format` during T1. Folded
  into the T1 commit rather than spawning a separate `chore(fmt)` commit
  since the drift was unrelated to but co-discovered with the Q9 work;
  flagged in the T1 commit body.
